1. What is .gitignore? You can write the answer either in Korean or English.
	.gitignore는 Git에 의한 버전 관리가 필요하지 않은 파일들을 지정하여 Git이 버전 관리를 할 때 이 파일들을 무시하도록 해 주는 파일이다.
	따라서 .gitignore 파일에 작성한 파일이나 디렉토리는 계속 untracked 상태로 남아있게 된다. 
	그러나 이미 tracked 상태였던 파일을 .gitignore에 추가했을 때는 무시되지 않으므로 이런 파일들은 더 이상 Git에서 track하지 않도록 만든 뒤 .gitignore에 추가해 주어야 한다.
2. Why do Github users need an SSH key pair? (Users can use either SSH key pair or github account info.) Please write down a brief explanation of SSH key
	변경사항을 반영하기 위해 github과 같은 원격 저장소에 접근할 경우 인증을 요구하는데, 이 때 인증방식으로 SSH를 사용하면 매번 github account info를 입력하지 않고도 github에 연결할 수 있다.
	SSH 인증을 위해서는 SSH key pair (public key과 private key)가 필요하다. 
	SSH 키(Key)는 공개키(public key)와 개인키(private key)로 이루어진다. 
	개인키는 로컬에 위치해야 하고, 공개키는 서버에 등록해야 한다. 
	클라이언트가 서버에 SSH연결을 요청하면 서버는 random string을 생성하여 클라이언트에게 보낸다. 
	클라이언트는 전달받은 random string을 개인키로 암호화하여 서버에게 보내고, 서버는 받은 암호를 등록된 공개키로 해석한 후 2번에서 보냈던 random string과 일치하는지 확인한다. 일치한다면 인증이 완료된다.
